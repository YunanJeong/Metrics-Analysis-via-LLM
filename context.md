# 📊 Metric-AI-Reporter 프로젝트 컨텍스트 (2026-03-12)

이 문서는 프로젝트의 현재 상태, 주요 설계 결정 및 코드 수정 내역을 기록하여 향후 작업 시 빠른 맥락 파악을 돕기 위해 작성되었습니다.

---

## 🎯 프로젝트 개요
Prometheus 메트릭을 수집하여 AI(LLM)가 SRE 관점에서 분석하고, 그 결과를 리포트하는 자동화 시스템입니다. 특히 **로컬 LLM(9b급)**의 제한된 성능과 VRAM 환경에서도 안정적으로 동작하도록 최적화되었습니다.

## 🛠 주요 설계 결정 (Core Decisions)

### 1. 토큰 최적화 (Token Efficiency)
- **1노드 1줄 포맷**: 기존의 장황한 리스트 형식을 파이프(`|`) 구분 기반의 단일 행으로 압축하여 토큰 사용량을 약 **70% 절감**.
- **통합 식별자(ID) 도입**: `Source/Job/Instance`를 하나의 필드로 통합하여 데이터 밀도를 높이고 AI의 컨텍스트 유지력 향상.
- **증감폭(diff) 중심**: 어제 데이터를 통째로 보내는 대신, 현재 값 옆에 `(+n)` 또는 `(-n)` 형태의 24시간 전 대비 변화량만 표시.

### 2. 멀티 소스 지원 (Multi-Source Support)
- **PROM_TARGETS**: `0.env`에서 여러 Prometheus 서버를 `라벨|URL` 형식으로 유연하게 설정 가능.
- **소스별 세그멘테이션**: 데이터 가독성 및 AI 분석 정확도를 위해 소스(Source) 단위마다 빈 줄(Empty line)을 추가하여 논리적 경계 명시.

### 3. 지표 계산 정합성
- **Mbps 표준 준수**: 네트워크 속도는 SI 표준(1,000,000 bits)을 기준으로 계산하며, `irate`를 사용하여 실시간 트래픽 변화를 민감하게 반영.
- **시각적 힌트**: 리소스 사용량 80%(`!`), 90%(`!!`) 임계값 도달 시 기호를 자동 부착하여 AI의 사고 과정(Reasoning) 단축.

## 📂 파일별 상태 요약

### `1.fetch.sh` (핵심 수집기)
- **ID 통합 수정 완료**: `@SOURCE` 출력 라인을 제거하고 각 행의 앞에 통합 식별자 배치.
- **Mbps 계산 검증**: `irate` 결과값을 Bit 단위 환산 후 10^6으로 나누는 로직의 정확성 확인.

### `2.brain.sh` (AI 분석기)
- **9b 모델 생존 전략**: 추론 과정을 생략하고 기계적인 단답형 응답을 강제하는 프롬프트 적용.
- **멀티 벤더 지원**: Ollama(Local) 및 OpenAI API 구조 모두 지원.

### `README.md`
- 통합 ID 명세(`Source/Job/Instance`) 및 네트워크 단위 기준 반영.
